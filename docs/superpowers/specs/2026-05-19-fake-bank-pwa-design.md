# fake_bank iPhone 离线静态页面设计文档

## 目标

在 `C:\workSpace\fake_bank` 下实现一套可在 iPhone 16 Pro 上通过 Safari 添加到主屏幕、并可离线打开的静态 PWA 页面。页面视觉尽量 1:1 高仿参考截图，不包含 iPhone 顶部系统状态栏。

本项目只做前端静态页面与本地数据驱动，不依赖后端接口。

## 页面范围

共 4 个页面：

1. `index.html`
   对应图1“我的贷款”
2. `loan-detail.html`
   对应图2“贷款详情”
3. `usage-record.html`
   对应图3/图5“使用记录”
4. `repayment-detail.html`
   对应图4/图6/图7/图8/图9“还款详情”

## 页面跳转

固定跳转关系如下：

1. 图1中点击“个人住房贷款”右侧箭头，跳转到图2
2. 图2中点击“使用记录”，跳转到图3
3. 图2中点击“还款详情”，跳转到图4
4. 各页面左上返回按钮返回上一页

## 视觉约束

- 不渲染顶部系统时间、电量、5G、静音等状态栏
- 页面主体从导航栏开始高仿
- 颜色、字号、圆角、分隔线、留白、卡片间距尽量贴近截图
- 图标采用“截图裁切 + 必要时 SVG 重绘”的混合方案
- 图2在“还款详情”按钮以下不渲染任何额外内容

## 运行形态

页面实现为离线 PWA：

- 提供 `manifest.webmanifest`
- 提供 `service worker`
- 首次打开后缓存 `html/css/js/json/图片/图标`
- 从 iPhone 主屏启动后可离线访问

本项目不依赖任何在线接口。

## 数据来源

数据分为两类：

1. 固定贷款信息
   由本地 JS/JSON 文件维护
2. 还款计划和还款明细
   由 Node 脚本根据规则生成本地 JSON

## 已确认的数据口径

### 编号口径

- 图1中的合同相关编号按截图显示：`22810045400029440`
- 图4中的贷款合约号按截图和还款明细口径显示：`22810157800034988`
- 不强行统一为同一个编号

### 贷款信息口径

- 总额度：`1,300,000.00`
- 已用额度：`1,300,000.00`
- 可用额度：`0.00`
- 年化利率：`3.2%`
- 月供：`5715.71`

### 提前还款规则

- 提前还款日期：`2026-05-25`
- 提前还款金额：`500000.00`
- 首期正常还款日期：`2026-06-01`
- 提前还款后，月供保持不变
- 提前还款后，缩短总期数
- 不再强行固定到 `2038-10-01`

### 动态计算规则

- 图1“未还本金总计”按当前系统日期动态计算
- 图2“未还本金”按当前系统日期动态计算
- 动态计算逻辑需计入：
  - `2026-05-25` 的提前还款
  - 当前日期之前已发生的常规月供本金归还
- 若当前日期早于 `2026-05-25`，则不应扣除提前还款的 `500000`

### 固定为 0 的字段

以下字段以后都固定展示为 `0.00`：

- 罚息
- 罚息复利
- 图4中的“还罚息”
- 图4中的“还复利”

## 数据生成规则

### 还款计划重算

需要新增 Node 生成逻辑，基于以下事实重新生成计划：

- 原剩余本金基数：`1199601.82`
- 提前还款 `500000.00` 于 `2026-05-25` 生效
- 提前还款后剩余本金：`699601.82`
- 从 `2026-06-01` 开始按月供 `5715.71` 继续还款
- 月利率：`0.032 / 12`
- 每月利息 = 当期剩余本金 * 月利率
- 每月本金 = 月供 - 当期利息
- 最后一期按剩余本金结清

### 页面时间口径

页面中需要区分两种时间：

1. 记录本身的业务时间
   由数据决定
2. 页面筛选和未还本金计算基准时间
   由用户设备当前系统时间决定

## 错误处理

- 本地 JSON 加载失败时，页面展示轻量错误占位，不出现空白白屏
- 筛选结果为空时，展示空状态文案
- 设备时间早于任何还款记录时，图3和图4允许出现空列表

## 测试与验收

验收标准如下：

1. 四个页面跳转关系正确
2. 页面可离线访问
3. 能添加到 iPhone 主屏幕
4. 图2不渲染“还款详情”按钮以下内容
5. 图3默认收起，且上滑每次加载一笔
6. 图3不出现提前还款 `500000`
7. 图4按当前系统时间正确筛选近三月/近一年
8. 图1与图2中的未还本金按当前系统日期动态变化
9. 罚息、罚息复利相关字段始终为 `0.00`

## 已知限制

- 这是一套静态高仿页面，不接真实银行系统
- 图标若直接裁切不清晰，将改为 SVG 重绘近似
- “筛选”功能先做轻量静态交互，不模拟真实银行 App 的完整弹层体系
